# Mode 2: 术语优化

**用户诉求**：我剧本内容已经写好了，就想让术语规范一下——场景标题统一、转场词统一、时间术语统一。

**Skill 做什么**：只做**词典替换 + 规范化**，绝不改剧情内容。

## 触发关键词

- "规范术语"
- "统一场景标题"
- "把 CUT TO 改成中文"
- "把切换到统一成切至"
- "剧本术语大扫除"
- "统一内外景写法"
- "术语中英切换"

## 3 步流程

### Step 1: 扫描剧本

先跑一遍全文，找出：

**类别 1: 场景标题（Scene Heading）**
- 匹配模式：以 `内景/外景/INT./EXT./內景/外景/室内/室外/Inside/Outside/Interior/Exterior/内/外` 开头的行
- 提取三要素：内外景类型、地点、时间

**类别 2: 转场词（Transition）**
- 匹配模式：`切至/切换到/画面切换/转到/渐变/淡入/淡出/黑幕/画面黑掉/叠化/CUT TO/DISSOLVE/FADE/MATCH CUT/JUMP CUT/SMASH CUT/WIPE` 等
- 通常出现在场景末尾或段落末尾

**类别 3: 时间术语（Time of Day）**
- 匹配模式：`日/夜/白天/夜晚/晚上/上午/清晨/黄昏/傍晚/凌晨/DAY/NIGHT/MORNING/DAWN/DUSK/CONTINUOUS/LATER`

**类别 4: 括号内动作提示（Parenthetical）**
- 匹配模式：全部被 `()` / `（）` 包围的短句
- 规范化：`（低声地）→ （低声）`；`(smiling widely) → (smiling)`

### Step 2: 按目标语言 + 规范映射

`hub_read` 3 个共享词典：
- `references/vocab-scene-terms.md`
- `references/vocab-transitions.md`

对每个匹配项，按目标语言（用户指定 zh-CN / en-US / zh-TW / ja-JP / ko-KR）查表替换。

**示例映射**：

| 原文 | → 中文规范 | → 英文规范 |
|------|----------|----------|
| `1. 张家客厅，白天` | `内景. 张家客厅 - 日` | `INT. ZHANG'S LIVING ROOM - DAY` |
| `切换到：` | `切至：` | `CUT TO:` |
| `画面黑掉` | `淡出至黑。` | `FADE TO BLACK.` |
| `晚上，咖啡馆内` | `内景. 咖啡馆 - 夜` | `INT. CAFE - NIGHT` |
| `Cut to：` | `切至：` | `CUT TO:` |

### Step 3: 输出对照报告

**Before / After 对照表**：

```markdown
## 术语优化报告

**目标语言**: 简体中文
**扫描到**: 场景标题 12 处、转场词 8 处、时间术语 15 处

### 场景标题
| # | 原文 | 规范后 |
|---|------|--------|
| 1 | `1. 张家客厅，白天` | `内景. 张家客厅 - 日` |
| 2 | `室内 - 咖啡馆 - 傍晚` | `内景. 咖啡馆 - 黄昏` |
| ... | ... | ... |

### 转场词
| # | 原文 | 规范后 |
|---|------|--------|
| 1 | `切换到` | `切至：` |
| 2 | `画面黑掉` | `淡出至黑。` |
| ... | ... | ... |

### 时间术语（嵌在场景标题中的）
| # | 原文 | 规范后 |
|---|------|--------|
| 1 | `白天` | `日` |
| 2 | `深夜` | `夜` |
```

跟用户确认："这些改动是否都要应用？"

**用户回复**：
- **确认全部** → Skill 直接改文本输出
- **只应用某类** → 只应用用户指定类别（如只统一转场，场景标题不动）
- **有争议** → 让用户逐条确认（Skill 不擅自做决定）

## 特殊规则

### 规则 1: 不动"戏名 / 剧名 / 人名"

如果场景标题包含专有名词（人名、地名、剧名），只规范格式框架，专有名词一字不改：

```
原文: 1. 张家客厅，白天
后:  内景. 张家客厅 - 日   ✓（"张家客厅" 保留）
错:  内景. LIVING ROOM - 日   ✗（不许翻译地名）
```

### 规则 2: 中英混排时保留原文

如果一部剧本中英混用（比如中文剧本有几处英文场景标题），默认按用户指定的目标语言统一。除非用户明确说"保留双语"。

### 规则 3: 括号内提示轻改

`parenthetical` 只做**极简化**改动：
- 去掉赘词：`（他小心翼翼地）→ （小心翼翼）`
- 统一括号：`(笑)` → `（笑）`（中文剧本用全角）
- 移除主语：`（张三看着窗外）→ （看着窗外）`（因为紧跟 character 已经知道谁在说）

但**不动**具体情绪 / 动作描述本身，只清理格式。

### 规则 4: 转场留到必要时

**Skill 提醒**：好莱坞现代剧本里，`CUT TO:` 通常不写（默认硬切）。如果用户原文里到处都是 `切至：`，Skill 应该问一句：「大部分转场默认硬切，要不要只保留特殊转场（叠化 / 淡入淡出 / 匹配剪辑），删掉普通的切至？」

## Review Checkpoint

术语优化完成后必查：

- [ ] 场景标题都按 `<内外景> <地点> - <时间>` 结构
- [ ] 转场词全部映射到规范词典（不出现"切换到" / "转到" 等）
- [ ] 时间术语用目标语言一致
- [ ] 专有名词（人名/地名）没有被误改
- [ ] `parenthetical` 括号符号统一（中文全角 / 英文半角）
- [ ] 目标语言一致（不出现中英混用）

## 常见问题

- **同一地点写法不同**：如"张家客厅" / "张家的客厅" / "张宅客厅"。Skill 提醒但**不自动合并**，让用户决定统一到哪个。
- **转场词过多**：现代剧本一般只在需要视觉转场时写。Skill 主动提示是否删掉冗余的 `CUT TO:`。
- **时间术语中英混用**：用户说"我要中英双版"时，输出两份文件（中文版 + 英文版），不要在一份文件里混。
