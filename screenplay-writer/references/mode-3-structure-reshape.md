# Mode 3: 结构重塑

**用户诉求**：我故事写完了但感觉节奏散、结构乱，想按某个经典叙事结构（三幕式 / 救猫咪 / 英雄旅程 / 起承转合 / TV Hour）来重组。

**Skill 做什么**：分析现有故事 → 匹配目标结构 → 给出场次重排建议。**不写具体台词，只做结构层面的编排**。

## 触发关键词

- "按三幕式重组"
- "救猫咪 15 拍"
- "英雄旅程"
- "起承转合"
- "结构混乱"
- "节奏散"
- "帮我梳理故事结构"
- "TV Hour 结构"

## 5 种结构模板

### 1. 三幕式（Three Act）

**用途**：通用叙事骨架，几乎所有故事都能套。适合电影、短剧、剧本。

**8 个节拍**：

| # | 节拍 | 位置（占比） | 说明 |
|---|------|-----------|------|
| 1 | 开场状态 | 0-5% | 建立主角日常和世界观 |
| 2 | 诱发事件 | 5-15% | 打破日常的关键事件 |
| 3 | 第一幕转折 | 20-25% | 主角被迫踏上旅程 |
| 4 | 中点反转 | 45-55% | 情势逆转，主角以为的错了 |
| 5 | 危机 | 65-75% | 主角遭遇最大挫折 |
| 6 | 第二幕转折 | 75-80% | 主角找到新的解决方式 |
| 7 | 高潮 | 85-95% | 决战/最终对决 |
| 8 | 余波 | 95-100% | 新常态收尾 |

**多语言标题**：`三幕式 / Three Act / 三幕構成 / 3막 구조`

### 2. 商业片节拍（Save the Cat / 救猫咪）

**用途**：好莱坞商业片黄金公式，Blake Snyder 的 15 拍法。适合商业电影、类型片。

**15 个节拍**：

| # | 节拍 | 位置（占比） | 说明 |
|---|------|-----------|------|
| 1 | 开场画面 | 0-1% | 一张对比性的开场图 |
| 2 | 主题陈述 | 5% | 主角需要学到的教训 |
| 3 | 铺垫 | 1-10% | 主角当前生活的问题 |
| 4 | 催化剂 | 12% | 打破一切的事件 |
| 5 | 争辩 | 12-25% | 主角犹豫要不要行动 |
| 6 | 进入第二幕 | 25% | 主角决定行动 |
| 7 | 副线（B Story） | 30% | 引入次要情感线 |
| 8 | 游戏时间 | 30-55% | 主角享受新世界，观众"想看的画面" |
| 9 | 中点 | 55% | 假胜利 or 假失败 |
| 10 | 坏人逼近 | 55-75% | 反派/危机步步紧逼 |
| 11 | 一无所有 | 75% | 主角失去一切 |
| 12 | 黑夜灵魂 | 75-80% | 主角内心最低谷 |
| 13 | 进入第三幕 | 80% | 主角悟出真谛，重新行动 |
| 14 | 结局 | 80-99% | 决战 + 全部支线收束 |
| 15 | 终场画面 | 100% | 呼应开场画面的对比 |

**多语言标题**：`商业片节拍 / Commercial Beat Sheet / 商業片節拍 / 商業映画ビート / 상업영화 비트`

### 3. 英雄旅程（Hero's Journey）

**用途**：Joseph Campbell 单一神话论。适合冒险 / 成长 / 奇幻题材。

**12 个阶段**：

| # | 阶段 | 说明 |
|---|------|------|
| 1 | 日常世界 | 主角熟悉的生活 |
| 2 | 冒险召唤 | 打破平静的召唤 |
| 3 | 拒绝召唤 | 主角一开始拒绝 |
| 4 | 导师 | 遇到智者/引路人 |
| 5 | 跨越门槛 | 主角踏入未知 |
| 6 | 试炼 / 盟友 / 敌人 | 经历考验，遇朋友和敌人 |
| 7 | 接近深层洞穴 | 逼近核心危险 |
| 8 | 严峻考验 | 主角面临最深的挑战 |
| 9 | 奖赏 | 获得核心宝物或觉悟 |
| 10 | 归途 | 返回日常世界 |
| 11 | 复活 | 最后一次考验，脱胎换骨 |
| 12 | 带着灵药归来 | 用所得改变世界 |

**多语言标题**：`英雄旅程 / Hero's Journey / 英雄旅程 / 英雄の旅 / 영웅의 여정`

### 4. 起承转合（Kishotenketsu）

**用途**：东方叙事传统，不依赖冲突而依赖变化。适合日剧、艺术电影、散文式叙事。

**4 个阶段**：

| # | 阶段 | 说明 |
|---|------|------|
| 起（Ki） | 建立人物与世界 | 铺陈状态 |
| 承（Sho） | 深化关系与规则 | 拓展 |
| 转（Ten） | 引入反差或新视角 | 关键转折 |
| 合（Ketsu） | 重组意义并收束 | 全新理解 |

**多语言标题**：`起承转合 / Kishotenketsu / 起承轉合 / 起承転結 / 기승전결`

### 5. 一小时剧集（One-Hour TV）

**用途**：美剧标准 7 幕结构（含中间广告插播位）。适合网剧、周播剧。

**7 个幕**：

| # | 幕 | 说明 |
|---|-----|------|
| 1 | 冷开场（Cold Open） | 播出前的钩子片段 |
| 2 | 第一幕问题 | 引出本集核心冲突 |
| 3 | 第二幕升级 | 冲突加剧 |
| 4 | 第三幕反转 | 剧情反转 |
| 5 | 第四幕代价 | 主角付出代价 |
| 6 | 第五幕解决 | 本集问题解决 |
| 7 | 尾声钩子（Tag） | 抛出下集悬念 |

**多语言标题**：`一小时剧集 / One-Hour TV / 一小時劇集 / 一時間ドラマ / 1시간 드라마`

## 3 步流程

### Step 1: 让用户选目标结构

Skill 反问：「你想按哪个结构重组？」列出 5 个选项 + 简短说明，让用户选。

如果用户已经指定了（"按三幕式"），跳过此步直接进 Step 2。

### Step 2: 分析用户现有故事

对用户已有故事文本或梗概：
1. **拆分场次** — 用 Mode 1 的元素识别，找到所有 `scene`
2. **给每场打标签** — 内容概括 + 情感基调 + 剧情作用（是铺垫？转折？还是高潮？）
3. **对齐目标模板** — 看每个场次落在目标模板的哪个节拍位置
4. **诊断问题**：
   - 哪些节拍**缺失**？（比如没有诱发事件）
   - 哪些节拍**位置不对**？（中点反转出现太早/太晚）
   - 哪些场次**冗余**？（跟节拍无关的场次）

### Step 3: 输出重组建议

**结构诊断报告**：

```markdown
## 结构诊断报告

**目标结构**：三幕式（8 拍）
**现有场次数**：42
**建议场次数**：40-50（符合三幕式规模）

### 节拍对齐表

| # | 节拍 | 期望位置 | 你的对应场次 | 状态 |
|---|------|--------|------------|------|
| 1 | 开场状态 | 场次 1-2 | Sc.1 | ✅ 到位 |
| 2 | 诱发事件 | 场次 3-6 | 无 | ❌ 缺失 |
| 3 | 第一幕转折 | 场次 8-10 | Sc.5（提前了） | ⚠️ 位置不对 |
| 4 | 中点反转 | 场次 18-22 | Sc.19 | ✅ 到位 |
| ... | ... | ... | ... | ... |

### 建议改动

1. **补一个诱发事件场次**（在 Sc.2 和 Sc.5 之间）
   - 建议内容：<主角>接到那个关键电话/见到关键人物/发生打破日常的事件
   - 场次长度：1-2 场
   
2. **第一幕转折 Sc.5 位置太早**
   - 建议：把 Sc.5 移到 Sc.8-10 附近
   - 或者：在中间补 2-3 场作为过渡

3. **删除冗余场次**：Sc.14, Sc.27, Sc.35（这 3 场跟主线关联弱）

### 场次重排后的目录

（Skill 输出重排后的场次列表，只调顺序 + 增删场次，不写具体台词）
```

### Step 4: 可选 - 应用改动

用户看完诊断报告后：
- **只要诊断** → Skill 就此停手，用户自己判断怎么改
- **应用重排** → Skill 直接调整原剧本的场次顺序 + 加空场次占位符
- **重新出剧本** → 用户手动填空场次占位符的内容（Skill **不代写台词**）

## 特殊规则

### 规则 1: 不代写剧情

**Skill 铁律**：只做结构层面的建议（"这里缺一个诱发事件"），**不写具体情节**（"张三接到分手电话，然后…"）。

如果用户说"帮我把诱发事件补上"，明确回复：**结构重塑不代写剧情。诱发事件的具体内容需要你自己构思，我可以帮你判断这个事件是否符合诱发事件的节拍功能。**

### 规则 2: 尊重原作

如果用户原剧本已经明显偏离目标结构（比如是散文诗式的），Skill 应该提示：**你的故事更接近起承转合结构，硬套三幕式可能破坏原有节奏，建议改用起承转合。**

### 规则 3: 场次数量建议

不同结构对应不同场次规模：

| 结构 | 建议场次数 |
|------|-----------|
| 三幕式 | 40-60 |
| 救猫咪 | 40 |
| 英雄旅程 | 50-80 |
| 起承转合 | 20-40 |
| TV Hour | 20-30 |

跟用户实际场次数偏差过大时提示。

## Review Checkpoint

结构重塑输出后必查：

- [ ] 节拍点数量 = 目标模板节拍数
- [ ] 每个节拍点有对应的场次（或明确标为"待补"）
- [ ] 场次总数在合理范围
- [ ] 主角弧线连续（不出现主角性格断层）
- [ ] 支线（如 B Story）跟主线有交汇点

## 常见问题

- **用户觉得诊断不对**：Skill 承认自己只是模式化对齐，让用户手动指定哪些场次对应哪个节拍
- **多主角故事**：目前只支持单主角。多主角先选主要视角，其他角色作为辅助线
- **系列剧 / 长篇**：单集用 TV Hour；全季用三幕式（每季一个大三幕）
